# משימות הצוות — Task PWA (UX Upgrade)

Next.js 14 (App Router) + Tailwind + Framer Motion + Supabase, fully RTL, built for 5 people using it daily on their phones.

## הרצה מקומית

```bash
npm install
cp .env.local.example .env.local   # מלאו את פרטי Supabase שלכם
npm run dev
```

פתחו `http://localhost:3000` — תגיעו למסך בחירת תפקיד להדגמה (בפרודקשן זה יוחלף בהתחברות אמיתית דרך Supabase Auth).

- `/staff?user=noam` — דשבורד עובד/ת (זהות ההדגמה נקבעת ע"י `?user=`)
- `/manager` — דשבורד מנהל/ת

## מה נבנה כאן

### 1. עיצוב ותמה
- פלטת בסיס Slate/Zinc רכה, אקסנט Indigo, וצבעי סטטוס פסטליים (`tailwind.config.ts`).
- Dark Mode אוטומטי לפי הגדרות הטלפון (`darkMode: "media"`), ללא צורך במתג ידני.
- פונט Rubik — תמיכה מלאה בעברית, עגול וחם, נטען דרך `next/font/google`.
- אייקונים מ-Lucide-React בכל מקום: סטטוסים, פעולות, כפתורים.

### 2. דשבורד עובד — "אפס חיכוך" (`app/staff/page.tsx`, `components/TaskCard.tsx`)
- כרטיס משימה עם **צ'קבוקס גדול ומספק** *וגם* מחווה של **החלקה ימינה להשלמה** (Framer Motion `drag`), כדי לכסות גם משתמשים שמעדיפים הקשה וגם כאלה שמעדיפים סווייפ.
- **קונפטי עדין** (`canvas-confetti` דרך `hooks/useConfetti.ts`) בכל השלמת משימה — מכבד `prefers-reduced-motion`.
- **מסך ריק מעודד** (`components/EmptyState.tsx`) כשאין משימות פתוחות.

### 3. דשבורד מנהל — שקט נפשי ויזואלי (`app/manager/page.tsx`)
- **טבעות התקדמות** (`components/ProgressRing.tsx`) — כללי ולפי כל איש צוות, במקום טבלת מספרים יבשה.
- **דגלים אדומים** למשימות שחרגו מדדליין — פועם בעדינות (`animate-pulse-soft`), לא צורח.
- **יומן פעילות** כציר זמן אנכי עם אווטארים וטקסט ידידותי (`components/ActivityTimeline.tsx`).

### 4. יצירת משימה (`components/CreateTaskSheet.tsx`)
- **Bottom Sheet** נגרר (Framer Motion), לא מודאל נוקשה — אפשר לסגור גם בגרירה למטה.
- **שיוך מהיר**: 5 כפתורים עגולים עם ראשי תיבות/צבע לכל איש צוות, בחירה בלחיצה אחת.

## חיבור ל-Supabase ולאוטומציות הטלפון

הפרונט הזה בנוי לעבוד מול הסכמה שמתוארת ב-`lib/supabase.ts` (טבלאות `team_members`, `tasks`, `activity_log` + RLS שמפריד בין הרשאות מפקדת לחברי צוות).

הפעולות הבאות בקוד הן בדיוק הטריגרים שהאוטומציה ב-Make.com + Twilio (שתוכננה במסמך התוכנית) צריכה להאזין להם:

- `handleComplete` ב-`app/staff/page.tsx` → עדכון `status = 'done'` ב-DB → מפעיל את אוטומציית "השלמת משימה" ששולחת WhatsApp/SMS למפקדת.
- בדיקת `isOverdue()` (`lib/utils.ts`) משוכפלת בצד השרת ע"י ה-Scheduled Scenario היומי ב-Make, שסורק `deadline < today AND status != 'done'` ושולח התראת חריגה לאחראי ולמפקדת.

כדי לחבר בפועל: יש להחליף את הנתונים המדומים ב-`lib/mock-data.ts` בקריאות אמיתיות דרך `lib/supabase.ts` (`supabase.from('tasks').select()/update()/insert()`), ולהוסיף Supabase Auth כדי שה-RLS יזהה מי מחובר.

## עדכון: סמל היחידה, היסטוריה והרשאות

- **סמל היחידה** — הועלה ל-`public/logo.png` ומוצג ב-`components/AppHeader.tsx` (כל מסך) וב-`app/page.tsx` (מסך הכניסה), עם פס תלת-צבעי עדין (`unit-red` / שחור / `unit-green`) שנגזר מהסמל, כתוספת עיצובית מרוסנת. `manifest.json` ואייקוני ה-PWA (`icon-192.png`, `icon-512.png`) מצביעים גם הם על הסמל — כדאי בהמשך להחליף אותם בגרסאות שנחתכו בדיוק ל-192×192 ו-512×512 פיקסלים לאיכות מיטבית באייקון שעל המסך הראשי.
- **היסטוריה לפי תאריכים/חודשים** — טאב חדש בתחתית המסך (`components/BottomNav.tsx`) עובר בין "משימות" (היום, פתוחות) ל"היסטוריה" (`app/staff/history`, `app/manager/history`), עם דפדוף בין חודשים (`components/MonthSwitcher.tsx`) ותצוגה מקובצת לפי יום (`components/MonthHistory.tsx`, לפי שדה `deadline`).
- **הרשאות** — נאכף כרגע בצד הלקוח: דף `/staff` מסנן `TASKS` לפי `assigneeId === userId` בלבד (חייל רואה רק את שלו, גם בתצוגת ההיום וגם בהיסטוריה), ואילו `/manager` מקבל את כל המערך ללא סינון. כשמחברים Supabase Auth בפועל — **חובה** להוסיף גם את מדיניות ה-RLS שמתוארת ב-`lib/supabase.ts`, כי סינון בצד הלקוח בלבד אינו הגנה אמיתית ברגע שיש נתונים אמיתיים וחיבור API.

## עדכון: חובר בסיס נתונים אמיתי (Supabase)

האפליקציה כבר לא עובדת על נתונים בזיכרון — היא מחוברת לפרויקט Supabase אמיתי:

- **סנכרון חי** — `lib/store.tsx` מאזין ל-Realtime של Supabase על כל הטבלאות (`team_members`, `tasks`, `task_comments`, `activity_log`). כשמישהו יוצר/מעדכן/מוחק משימה, כל מי שפתוח באפליקציה (בכל טלפון) רואה את זה מיידית, בלי לרענן.
- **אין יותר משימות דמו** — טבלת `tasks` מתחילה ריקה; רק חברי הצוות האמיתיים (עם המספרים האמיתיים) הוזנו מראש לטבלת `team_members`.
- **משתני הסביבה** — `.env.local` המקומי וגם משתני הסביבה ב-Netlify כבר מוגדרים עם ה-URL וה-anon key של הפרויקט.

### ⚠️ מגבלת אבטחה ידועה
מדיניות ה-RLS (Row Level Security) שהוגדרה כרגע היא **פתוחה** — כל מי שמחזיק את ה-anon key (שממילא חשוף בצד הלקוח) יכול לקרוא ולכתוב לכל הטבלאות. זה תואם בדיוק את מה שהיה קודם (בדיקת הרשאות בצד הלקוח בלבד), רק עכשיו עם נתונים אמיתיים ומשותפים. ההרשאות שאתם רואים באפליקציה (חייל רואה רק את שלו, מפקדת רואה הכל) **נאכפות רק בממשק**, לא בשרת. כדי לאכוף את זה באמת ברמת בסיס הנתונים, צריך להחליף את מסך ההתחברות הנוכחי (טלפון + קוד מדומה) ב-Supabase Auth אמיתי (OTP לפי טלפון), ואז לשנות את מדיניות ה-RLS כך שתבדוק `auth.uid()` מול `assignee_id` / `is_manager` — בדיוק כמו שהיה בטיוטה המקורית של הסכמה.
